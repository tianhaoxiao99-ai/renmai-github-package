# 软著提交代码页选择建议（桌面版）

## 1. 适用范围

本建议仅针对当前仓库中的仁迈 Windows 桌面版整理，不再按网页端、公开聊天或 Supabase 口径选取代码。

当前桌面版核心代码主要集中在以下区域：

- `lib/`
- `test/`
- `windows/`

## 2. 选码原则

- 优先体现桌面端入口、页面结构、聊天记录导入、关系报告、联系人洞察、AI 设置和本地存储。
- 优先选择当前主链路代码，不优先提交旧网页脚本、旧云端接口、截图文件和临时文件。
- 不建议把 `vendor/`、`.tmp-*`、`.log`、`.pid`、截图图片和旧版网页说明文档作为核心源码提交材料。

## 5. 代表性代码挑选优先级

### A. 一级优先

- `lib/providers/analysis_provider.dart`
- `lib/services/import_service.dart`
- `lib/services/local_report_service.dart`
- `lib/services/api_service.dart`
- `lib/screens/home/home_screen.dart`
- `lib/screens/import/import_screen.dart`
- `lib/screens/report/report_screen.dart`

这部分最能体现当前成品的核心能力：

- 聊天记录导入
- 关系评定
- 证据提炼
- 礼物建议
- AI 增强
- 引导式桌面工作台

### B. 二级优先

- `lib/models/comparison_report.dart`
- `lib/models/conversation_record.dart`
- `lib/providers/relationship_provider.dart`
- `lib/providers/display_settings_provider.dart`
- `lib/widgets/workspace_shell.dart`
- `lib/config/app_theme.dart`

这部分用于体现：

- 数据结构设计
- 派生分析逻辑
- 主题与可读性设计
- 工作台化交互组织

### C. 三级优先

- `test/import_service_test.dart`
- `test/local_report_service_test.dart`
- `test/widget_test.dart`

这部分用于体现：

- 导入解析验证
- 报告生成验证
- 应用启动冒烟验证

## 6. 当前不建议优先提交的内容

- `public-chat.js`
- `index.html`
- `functions/`
- `supabase/`
- `vendor/`
- `softcopyright-output/`
- `softcopyright-package/`
- `archive/legacy_app/`
- 各类 `.tmp-*`、`.log`、`.pid`、截图文件

这些内容要么属于旧版网页实现，要么属于材料输出目录，不再代表当前桌面成品。

## 7. 自动整理建议

建议后续增加一份桌面版源码汇编脚本，专门按上面的顺序导出：

- 入口与路由
- 模型与 Provider
- 服务层
- 主页面
- 测试文件

这样能避免继续沿用旧网站版的汇编脚本与页码建议。

## 8. 最终提交建议

当前桌面版软著材料建议按以下组合准备：

- 申请表：使用 [软著申请表填写稿](./软著申请表填写稿.md)
- 设计文档：`软件设计说明书.md`
- 使用文档：`使用说明书.md`
- 截图材料：使用 [软著截图清单](./软著截图清单.md)
- 源程序材料：按本文件的桌面版顺序汇编

## 9. 当前不建议继续沿用的旧口径

以下说法已经不适合本版成品：

- 网站版核心代码
- 公开账号、注册登录、实时聊天
- Supabase 数据表与 RLS 权限模型
- Web 入口页面与浏览器端脚本
