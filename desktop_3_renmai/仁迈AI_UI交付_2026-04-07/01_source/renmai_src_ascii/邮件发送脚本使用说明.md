# 邮件发送脚本使用说明

项目里已经提供了两个可直接使用的 PowerShell 脚本：

- `scripts/send-qq-mail.ps1`
- `scripts/send-project-update.ps1`

## 1. 准备 QQ 邮箱 SMTP 授权码

你需要先在 QQ 邮箱中开启 SMTP 服务，并获取授权码。

发件信息需要这两项：

- QQ 发件邮箱，例如 `123456789@qq.com`
- QQ 邮箱 SMTP 授权码

## 2. 创建本地配置文件

把 `scripts/mail.local.example.ps1` 复制为 `scripts/mail.local.ps1`，然后填入你自己的信息：

```powershell
$env:QQ_SMTP_USER = '你的QQ邮箱@qq.com'
$env:QQ_SMTP_AUTH_CODE = '你的QQ邮箱SMTP授权码'
$env:QQ_SMTP_DEFAULT_TO = '1943358991@qq.com'
```

说明：

- `scripts/mail.local.ps1` 已加入 `.gitignore`
- 这个文件只保存在你本机，不会被提交到仓库

## 3. 先做一次干跑检查

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\send-project-update.ps1 -DryRun
```

如果输出了 `SMTP / From / To / Subject / BodyLen`，说明脚本结构正常。

## 4. 发送默认项目进度邮件

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\send-project-update.ps1 -Stage 'App 和 Web 阶段完成'
```

默认收件人是 `1943358991@qq.com`。

## 5. 使用摘要文件发送

先准备一个文本文件，例如 `summary.txt`：

```text
- Web 端 UI 已完成
- App 端欢迎页与登录链路已完成
- 下一步准备软著材料整理
```

然后执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\send-project-update.ps1 -Stage '阶段完成' -SummaryFile .\summary.txt
```

## 6. 发送自定义邮件

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\send-qq-mail.ps1 -Subject '项目完成通知' -Body '项目已经完成，请查收。'
```

## 7. 发送带附件的邮件

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\send-project-update.ps1 -Stage '提交前检查' -Attachments '.\软件设计说明书.md','.\使用说明书.md'
```

## 8. 当前项目内相关文件

- `scripts/send-qq-mail.ps1`：通用发信脚本
- `scripts/send-project-update.ps1`：项目状态通知脚本
- `scripts/mail.local.example.ps1`：本地配置示例
- `scripts/templates/project-update-body.txt`：默认邮件正文模板

## 9. 注意事项

- 真正发送前必须提供有效的 QQ 邮箱 SMTP 授权码
- 如果授权码错误，脚本会在 SMTP 发送阶段失败
- 如果只是想验证参数是否正确，优先使用 `-DryRun`
