# 聊天记录准备指南

这份指南是给第一次使用仁迈、也不熟悉电脑操作的人准备的。

## 一句话先说明白

仁迈当前最适合读取的是：

- `.txt`
- `.html`
- `.zip`
- 装着这些文件的文件夹

它暂时不直接读取：

- 微信或 QQ 的内部数据库
- 截图和录屏
- 看不懂的缓存文件

## 最推荐的入门方法

第一次测试时，不要一次导入全部历史聊天记录。  
请只选一位联系人，先准备一份能直接看懂的聊天记录文件。

## 微信用户怎么做

1. 先确定你要分析哪位联系人。
2. 如果你已经在电脑上有这段聊天记录，并且能整理成正常文字文件，优先保存为 `.txt` 或 `.html`。
3. 如果记录只在手机里，先通过微信自带的迁移/备份能力，把记录迁到电脑环境中。
4. 把需要分析的内容整理成可阅读文件。
5. 文件名尽量写成类似 `微信_小王.txt` 这种形式。
6. 回到仁迈导入。

## QQ 用户怎么做

1. 打开 Windows QQ。
2. 找到要分析的联系人聊天记录。
3. 尽量导出或另存为 `.txt` 或 `.html`。
4. 如果有多位联系人，可以放进同一个文件夹。
5. 也可以把多个文件压缩成 `.zip`。
6. 回到仁迈导入。

## 在仁迈里怎么导入

### 你手里是文件夹

1. 打开仁迈
2. 进入“导入向导”
3. 点击“选择导出目录”
4. 选中文件夹
5. 点击“开始导入”

### 你手里是几个文件

1. 打开仁迈
2. 进入“导入向导”
3. 点击“选择记录文件”
4. 选中 `.txt`、`.html` 或 `.zip`
5. 点击“开始导入”

## 什么样的文件最稳

最容易识别的是这种结构：

```text
[2026-03-20 10:00:00] 我: 生日快乐
[2026-03-20 10:01:00] 小王: 谢谢你
```

也就是：

- 有时间
- 有发言人
- 有正文

## 如果导入失败，先检查这几件事

1. 你是不是选错了目录
2. 文件后缀是不是 `.txt`、`.html`、`.zip`
3. 文件打开后能不能看懂
4. 文件里是不是有时间和发言人
5. 你导入的是不是截图、数据库、缓存

## 推荐做法

- 先从 1 位联系人开始
- 先从 1 份文件开始
- 成功后再批量导入
- 文件名里尽量写联系人名字

## 暂时不要这样做

- 不要直接把微信数据库丢进来
- 不要只导入聊天截图
- 不要一上来导入特别大的混合备份包
- 不要导入自己都看不懂的乱码文件
