# 仁迈 V1.0.0 软件设计说明书

## 1. 文档概述

### 1.1 编写目的

本文档用于说明“仁迈”软件的总体结构、模块划分、数据模型、关键处理流程、存储方式与安全设计，为软件开发、测试、维护及软件著作权申报提供技术说明依据。

### 1.2 文档范围

本文档对应版本为 `V1.0.0`，描述对象为基于 Flutter 构建的 Windows 桌面应用软件。内容聚焦于：

- 本地聊天记录导入
- 联系人关系分析
- 送礼建议生成
- 自定义 AI 接口增强
- 本地存储与隐私控制

## 2. 软件概述

### 2.1 基本信息

| 项目 | 内容 |
| --- | --- |
| 软件名称 | 仁迈 |
| 软件类型 | Windows 桌面应用 |
| 开发语言 | Dart |
| 开发框架 | Flutter |
| 功能定位 | 聊天记录整理与关系分析软件 |
| 当前版本 | V1.0.0 |

### 2.2 建设目标

1. 支持用户主动导入聊天记录文件并完成本地整理。  
2. 将非结构化聊天文本转换为可统计、可分析的标准化数据。  
3. 对联系人互动情况进行归类、评估和结果输出。  
4. 在本地分析基础上提供可选的 AI 增强能力。  

## 3. 总体架构

### 3.1 分层设计

系统整体采用分层设计，主要由界面层、状态管理层、服务层、数据模型层与存储层构成：

```text
┌─────────────────────────────────────────┐
│           界面展示层（Screens）          │
├─────────────────────────────────────────┤
│          状态管理层（Providers）         │
├─────────────────────────────────────────┤
│      本地分析与 AI 编排层（Services）    │
├─────────────────────────────────────────┤
│         本地数据模型层（Models）         │
├─────────────────────────────────────────┤
│       本地存储与安全配置层（Storage）     │
└─────────────────────────────────────────┘
```

### 3.2 主要模块

- `仪表盘模块`：展示导入概况、联系人数量和数据摘要  
- `导入模块`：完成文件选择、扫描、来源识别与解析处理  
- `报告模块`：展示关系排序、风险提示、行动建议和礼物建议  
- `联系人洞察模块`：展示单个联系人的分析结果与证据片段  
- `AI 设置模块`：维护 Base URL、API Key、模型名称和 AI 开关

## 5. 关键处理流程

### 5.1 聊天记录导入流程

```text
用户选择目录或文件
  -> 扫描 TXT / HTML / ZIP
  -> 若为 ZIP 则解包到临时目录
  -> 识别微信 / QQ 来源
  -> 解析联系人、时间和消息内容
  -> 去重并标准化为 ConversationRecord
  -> 生成 ImportedPackage
  -> 刷新本地报告
```

### 5.2 本地规则分析流程

```text
读取标准化消息
  -> 按联系人聚合
  -> 统计消息量、活跃天数、最近互动时间
  -> 识别互动信号与风险关键词
  -> 计算关系分值与关系等级
  -> 生成送礼建议和行动建议
  -> 输出 ComparisonReport
```

### 5.3 AI 增强流程

```text
用户主动点击 AI 增强
  -> 读取本地规则报告与关键消息样本
  -> 组装结构化提示词
  -> 调用用户自定义 AI 接口
  -> 要求模型返回固定 JSON 结构
  -> 合并为新的 ComparisonReport
  -> 刷新界面
```

## 6. 功能模块设计

### 6.1 导入与解析模块

该模块负责接收用户选择的文件或目录，对输入内容进行扫描、来源判断、格式识别、临时解包与文本提取，并输出标准化消息记录。

主要功能包括：

- 文件与目录选择
- TXT / HTML / ZIP 扫描
- ZIP 临时解压
- 聊天来源识别
- 消息时间、发言人、正文提取
- 重复消息过滤

### 6.2 关系分析模块

该模块基于标准化消息记录进行联系人聚合、互动统计与规则判断，并输出联系人关系结果。

主要功能包括：

- 联系人维度聚合
- 活跃天数统计
- 最近互动时间计算
- 风险词与积极信号识别
- 关系等级评估
- 行动建议生成

### 6.3 礼物建议模块

该模块根据联系人关系阶段、互动内容和规则库生成礼物方向建议，作为综合报告的一部分输出。

### 6.4 AI 增强模块

该模块在用户主动启用的前提下，读取本地规则分析结果与关键样本数据，调用用户配置的外部模型接口，对本地报告进行补充说明和结构化增强。

## 7. 状态管理设计

### 7.1 `AnalysisProvider`

职责：
- 初始化本地工作区
- 管理导入记录、消息记录和综合报告
- 保存 AI 配置
- 触发本地规则分析与 AI 增强分析

### 7.2 `RelationshipProvider`

职责：
- 消费 `AnalysisProvider` 输出
- 对联系人洞察做筛选、检索与分类派生
- 供联系人列表与分布摘要使用

### 7.3 `GiftProvider`

职责：
- 消费综合报告中的礼物建议
- 维护当前礼物选择与历史记录

## 8. 本地存储设计

存储内容包括：
- AI 配置
- 已导入批次记录
- 标准化消息记录
- 最近一次综合报告
- 当前选中的联系人

存储方式：
- 普通结构化数据写入 `SharedPreferences`
- 敏感配置写入 `FlutterSecureStorage`

## 9. 非功能性设计

### 9.1 隐私设计

- 不承担人与人之间的实时社交沟通能力
- 默认仅在本地解析、整理和保存聊天内容
- 只有用户主动触发 AI 增强时才会发起外部请求
- 对外部模型接口的使用由用户自行配置并控制

### 9.2 可维护性设计

- 通过 `models / providers / services / screens` 实现清晰分层
- 将导入解析、规则分析、AI 调用拆分为独立服务
- 以结构化模型作为界面展示的数据来源

### 9.3 扩展性设计

- 可继续补充微信、QQ 等不同导出格式解析器
- 可扩展更多礼物模板和关系策略
- 可支持更多兼容通用接口规范的模型服务

## 10. 运行与部署说明

软件采用 Flutter 桌面应用方案实现，主要运行于 Windows 环境。应用以本地方式运行，聊天记录文件由用户从本地选择导入，分析结果与配置信息保存在本地存储中。对于 AI 增强能力，用户可按需配置外部服务地址、模型名称和访问密钥。
